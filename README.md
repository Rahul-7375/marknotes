# MarkNotes — Single Server Setup

One server. One port. Everything runs together.

## Quick Start

```bash
# 1. Install server dependencies
npm install

# 2. Build the React frontend
npm run build

# 3. Run everything
npm run dev
```

Open → **http://localhost:5000**

---

## How It Works

```
marknotes/
├── server.js        ← Express API + serves React build
├── db.js            ← SQLite setup (auto-created)
├── package.json     ← server dependencies
└── client/          ← React app
    ├── src/
    └── build/       ← generated by npm run build (gitignored)
```

- In **production**: `npm run build` compiles React into `client/build/`, then `npm run dev` serves it as static files from Express at `http://localhost:5000`.
- In **development**: run `npm run dev` for the API, and optionally `cd client && npm start` in a second terminal for React hot-reload at `http://localhost:3000` (proxied to port 5000).

## All Commands

| Command | What it does |
|---|---|
| `npm install` | Install server dependencies |
| `npm run build` | Install client deps + build React |
| `npm run dev` | Start Express server (port 5000) |
| `npm start` | Build + start in one shot |

## API

All endpoints are at `/api/...`

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/notes` | List notes (`?search=`, `?tag=`, `?page=`, `?limit=`) |
| POST | `/api/notes` | Create note |
| PUT | `/api/notes/:id` | Update note |
| DELETE | `/api/notes/:id` | Delete note |
| GET | `/api/notes/:id/versions` | Version history |
| POST | `/api/notes/:id/restore/:vId` | Restore version |
| GET | `/api/tags` | All tags |
